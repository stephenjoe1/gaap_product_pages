Deploy and run government apps

GOV.UK Platform as a Service (PaaS) is a quick, easy and cheap way to host government applications. It allows you to deploy and host your application in multiple languages and frameworks without building your own hosting infrastructure.

The PaaS manages all the necessary platform infrastructure for you – eg logging, monitoring, backing services, networking and high availability – so you can focus on building your digital services.

Benefits of GOV.UK PaaS

GOV.UK PaaS makes it easier for teams to deploy and run applications without having to build and manage their own infrastructure. This avoids unnecessary repetition across government – reducing the need for infrastructure and webops engineers to spend time solving the same problems.

Benefits to service teams include:

  • save time and money associated with setting up a hosting infrastructure
  • simple procurement process
  • accredited for information up to OFFICIAL
  • apps are not locked into a single cloud provider, so costs won’t escalate
  • source code is open
  • easy for development teams to manage their apps without support from the platform team
  • deploy and run applications written in a range of languages and web frameworks
  • rapidly scale app instances
  • deployed across multiple data centres, making it resilient
  • full platform-level support
  • built on an open source platform (Cloud Foundry) and deployed to a public cloud

How it works

Service teams manage the components that are unique to their applications. These are the user-facing front end apps, back end APIs, supporting backing services (like databases and caches), scheduled tasks and application security. It is also easy to ship logs from running apps to logging infrastructure of your choice.

The PaaS manages the platform components involved with hosting, such as infrastructure, platform logging and monitoring, provisioning backing services, securing the infrastructure, ensuring the platform is highly available.

Applications running on the platform are isolated from each other; they can’t read or change each other's code, data or logs.

How it’s built

GOV.UK PaaS is built using the open source Cloud Foundry project and runs on Amazon Web Services (AWS). Users don’t have to interact with AWS directly; all interactions required to deploy and manage your applications are carried out through Cloud Foundry commands.

The platform can host applications written in multiple languages, including JVM languages, Python, Go, Ruby, PHP, and many frameworks, such as Rails, Node.js and Django.

You can find out more about how the PaaS is built and how it works by reading the documentation.

Find out more about available backing services.

How you manage it

The PaaS is designed to be self-service, giving teams complete control over deploying, managing and changing their apps. This allows application teams to work and iterate quickly and means the platform team can focus exclusively on improving the platform itself.

To interact with the PaaS you use the Cloud Foundry command line. You can find out more about this by reading the documentation in the Quick Setup Guide.

Technical requirements

To be hosted by Government PaaS, your application must:

  • follow the twelve-factor application principles - this will be the case if your app was written to be deployed to another PaaS like Heroku
  • not require any backing service apart from a database (currently the only service available is PostgreSQL)
  • not carry data at SECRET or above (we are working towards accreditation for Personal data)
  • be written in a language supported by the default Cloud Foundry buildpacks

Support and cost

The platform will be supported 24/7 by GDS, although application teams will continue to support their individual applications.

GDS absorbs the cost of building the platform and we pass on the running costs: you simply pay for the hosting, compute resource and backing services you use.
